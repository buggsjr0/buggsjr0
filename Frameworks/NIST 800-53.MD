# NIST 800-53 <br/>

It's the massive catalog of every possible fire-prevention tool ever invented and the rules for which ones you must use.<br>
NIST 800-53 is the mater menu of security controls. It answers "What specific safeguards should we have in place?"<br>
- If buliding a fortress 800-37 says "you need a fortress"<br>
- 800-53 says "here's every possible wall, moat, guard tower and drawbridge you could install and which ones are required<br>
- 800-61 says "here is what to do when someone breaches the wall"


---

## Control Families
- 800-53 organizes controls into 20 families, each covering a different security domains:

| Abbreviation | Description |
| ----- | ----- | 
| AC| Access Control | | | SI | System & Info Integrity |
| AU | Audit & Accountability |
| AT | Awareness & Training |
| CM | Config Mgmt|
| CP | Contigency Planning |
| IA	 | Identification & Auth |
| IR | Incident Response |
| MA | Maintenance |
| MP | Media Protection|
| PT | PII Processing |

## Why This Is Critical for SOC Analysts<br>
This isn't just a government document, it's essentially the job description of a SOC analyst written out as a framework.

### 800-61 In Use
| SOC Activity | Where 800-61 Lives
| ----- | ----- |
| Monitoring SIEM alerts| Detection & Analysis |
| Triaging tickets by severity | Detection & Analysis |
| Isolating a compromised endpoint	 | Containment |
| Running malware scans & remediation | Eradication |
| Writing incident tickets & reports | Post-Incident |
| Building/updating runbooks	 | Preparation |
| Threat hunting | Detection & Analysis |

---
| Severity Tiers | (How SOCs Prioritize)
| ----- | ----- |
| 🔴  CRITICAL | Active breach, data exfiltration in progress → drop everything |
| 🟠  HIGH     | Ransomware, privileged account compromise → respond now |
| 🟡  MEDIUM   | Malware detected, contained → respond within hours |
| 🟢  LOW      | Policy violation, recon activity → handle in queue |

---

### Key Takeaway 
NIST 800-61 is the theoretical backbone behind everything a SOC analyst woud do on a shift. Open a ticket, isolate the host, escalate to a Tier 2 or write an after action report. All of these actions are executing this framework.
